# Politicians Tracker

A local, JSON-backed dashboard for public New Zealand political data. The app
tracks MPs, bills, voting records, expenses, opinion polls, Crown spending, and
related source-backed context for analysis.

The project is intentionally file-based: collectors normalize public sources into
versionable JSON under `data/`, and the Next.js app reads those files at request
time through server-side helpers.

## Quick Start

```bash
pnpm install
pnpm data:collect all
pnpm data:photos
pnpm data:enrich
pnpm dev
```

Open http://localhost:3000.

No secrets or private credentials are required. The data pipeline uses public
sources only.

## Requirements

- Node.js 20+

- Network access to public data sources when refreshing datasets

Use `pnpm check`, `pnpm typecheck`, and `pnpm test` before opening a pull

Biome intentionally excludes `data/`, `.cache/`, `public/`, CSS files,
`src/components/ui/`, and `pnpm-lock.yaml`. Do not rely on Biome to clean up

## App Architecture

The app uses the App Router. Most data reads should go through `src/lib/data.ts`,
which is marked `server-only` and returns an empty dataset when a JSON file is
missing. This keeps pages resilient during partial data refreshes.

Client components are used for interactive charts, filters, command palette,
chat, and browser-only behavior. Keep data loading and filesystem access on the
server side.

When adding routes, update the navigation, sitemap, metadata, and any relevant
tests or docs. Shared display formatting should live in `src/lib/utils.ts` so
SSR and browser rendering stay deterministic.

## Data Contract

Every primary dataset in `data/<domain>.json` uses this wrapper:

```ts
interface Dataset<T> {
   meta: {
      domain: Domain;
      count: number;
      collectedAt: string;
      sources: string[];
   };
   records: T[];
}
```

Record types and the `Domain` union live in `src/types/records.ts`. Canonical
domains are:

- `mps`
- `bills`
- `votes`
- `expenses`
- `ministers`
- `polls`
- `spending`

Every collected record should include `sourceUrl` and `fetchedAt` where the type
allows it. Preserve those fields when transforming or filtering data; they are
the audit trail for public-interest analysis.

`data/enrichment.json` is a secondary Wikipedia-derived dataset keyed by `mpId`.
It uses the same `Dataset<T>` wrapper, but it is not part of the `Domain` union
because it is optional enrichment rather than a primary collector domain.

## Managing Data

Run all primary collectors with:

```bash
pnpm data:collect all
```

Run one domain with:

```bash
pnpm data:collect mps
pnpm data:collect bills
pnpm data:collect votes
pnpm data:collect expenses
pnpm data:collect ministers
pnpm data:collect polls
pnpm data:collect spending
```

Most collectors cache fetched responses for 24 hours under `.cache/`. Use
`--force` when you need fresh source reads for collectors that pass the flag
through to the fetch helper:

```bash
pnpm data:collect all --force
pnpm data:collect mps --force
```

The generated `data/*.json` files are tracked. Commit them when the source data
has intentionally changed. `.cache/` is ignored and should not be committed.

### Recommended Refresh Flow

For a full public-data refresh:

```bash
pnpm data:collect all --force
pnpm data:photos --force
pnpm data:enrich --force
pnpm typecheck
pnpm test
```

Then inspect the dataset counts and any collector warnings before committing.
Warnings are common when a public source is temporarily unavailable or protected
by a browser challenge, but they should be understood rather than ignored.
If you need a completely cold refresh, delete `.cache/` first; the Treasury
spending collector currently relies on cache expiry for its workbook download.

For smaller updates:

- Update `mps` first when the current roster changes. `votes`, `expenses`,
   `data:photos`, and `data:enrich` depend on the MP roster and `mpId` values.
- Update `bills` and `votes` together when voted.nz division data changes.
- Update `expenses` after `mps` so spreadsheet names can be matched to known MPs.
- Update `polls` independently when Wikipedia polling tables change.
- Update `spending` when Treasury republishes the Fiscal Time Series workbook;
   if the workbook URL changes, update `scripts/collect/spending.ts`.

### Data Sources

| Dataset | Source | Notes |
| --- | --- | --- |
| `mps` | data.govt.nz `members-of-parliament` package | Official current members CSV from Parliamentary Service. `mpId` is a `surname-first` slug aligned with voted.nz. |
| `bills` | voted.nz divisions page | Bills with recorded personal votes. Introduced date is approximated from the earliest recorded division date. |
| `votes` | voted.nz MP person pages | Per-MP vote rows from each MP's "How X voted" section. Some current MPs have no voted.nz page and are skipped gracefully. |
| `expenses` | data.govt.nz `member-of-parliament-expenses` package | Recent quarterly XLSX files. The collector detects variable spreadsheet layouts and matches member names to `mps`. |
| `ministers` | DIA Ministers' expense releases | Best-effort HTML parsing. This source may require a real browser and can produce an empty dataset with a warning. |
| `polls` | Wikipedia parse API | Party-vote polling tables across recent election cycles. Source content is CC BY-SA 4.0. |
| `spending` | Treasury Fiscal Time Series XLSX | Annual Crown spending, functional categories, and nominal GDP in NZD millions. |
| `enrichment` | Wikipedia API | Optional MP biographies, descriptions, and portrait backfill data. |

Prefer official or machine-readable public sources. Current known good sources
include data.govt.nz CKAN APIs, voted.nz, Wikipedia APIs, and Treasury XLSX
files. Parliament.nz HTML pages are known to return bot-challenge content and
should not be scraped directly.

### Cache Behavior

Collector fetches go through `scripts/lib/fetch-cache.ts`:

- Text and JSON responses are cached as hashed `.txt` files.
- Binary resources such as XLSX files are cached as hashed `.bin` files.
- The default cache TTL is 24 hours.
- Live requests are spaced by a polite delay.
- A browser-like user agent is used for sources that reject generic fetches.

If a source looks stale, rerun the relevant collector with `--force` or delete
the relevant `.cache/` entries. Do not commit cache files.

### Photos and Enrichment

Portraits and biographies are managed separately from the primary collectors:

```bash
pnpm data:photos        # download portraits to public/politicians/<mpId>.jpg
pnpm data:photos --force
pnpm data:enrich        # write data/enrichment.json and backfill missing photos
pnpm data:enrich --force
```

Both scripts require `data/mps.json`. Wikipedia rate limits aggressively, so the
scripts include delays and retries. Existing matched enrichment records and
portrait files are skipped unless `--force` is passed.

### Matching Rules

- `mpId` is the stable join key for MPs, votes, expenses, photos, and enrichment.
- The MP collector derives `mpId` from the official `Surname, First` contact
   field, then applies fixes from `scripts/lib/mp-overrides.ts` for corrupted or
   diacritic-sensitive source names.
- Expense matching uses normalized name tokens so `First Last` and
   `Surname, First` forms can still match.
- Unmatched expense rows are retained with `mpId: null`; do not discard them
   unless the source row is invalid.

### Adding or Changing a Dataset

1. Add or update the record type in `src/types/records.ts`.
2. If it is a primary domain, add it to the `Domain` union.
3. Create `scripts/collect/<domain>.ts` exporting a `Collector`.
4. Register the collector in `scripts/collect/index.ts`.
5. Add a server-side loader or aggregation in `src/lib/data.ts`.
6. Add UI, tests, and docs for the new domain.
7. Run the collector and commit the updated `data/<domain>.json` when the change
    is intentional.

Keep schemas source-neutral. Do not rename fields to match one upstream file's
wording when the canonical type already has a normalized field.

### Data Quality and Ethics

- Use only public data.
- Preserve source URLs, fetch timestamps, and dataset source lists.
- Respect source licenses and attribution requirements, especially CC BY and CC
   BY-SA sources.
- Prefer structured APIs and downloadable files over HTML scraping.
- Treat blocked or browser-challenged sources as unavailable instead of trying
   to bypass protections.
- Keep analysis neutral and time-bounded. Money views should state their period
   coverage and avoid implying misconduct from raw totals alone.

## Testing

Tests live next to the code they cover, usually in `__tests__` directories. Jest
is configured with Next.js support, jsdom, Testing Library, and the `@/*` module
alias.

Useful commands:

```bash
pnpm test
pnpm test -- scripts/lib/__tests__/normalize.test.ts
pnpm test -- src/components/__tests__/stat-card.test.tsx
pnpm typecheck
pnpm build
```

When data code changes, add or update tests for normalization and aggregation
logic. When UI logic changes, prefer focused component tests for formatting,
fallbacks, and empty states.

## Common Gotchas

- `src/lib/data.ts` is server-only. Do not import it into client components.
- Keep filesystem access in server modules or scripts.
- Use deterministic date and number formatters for SSR-rendered client
   components; locale-dependent browser output can cause hydration mismatches.
- Recharts tooltip text can inherit series colors. Use explicit tooltip label
   and item styles when adding charts.
- Do not edit generated shadcn primitives casually; `src/components/ui/` is
   excluded from Biome to reduce generated churn.
- Do not scrape parliament.nz HTML pages directly; use machine-readable sources
   or documented public APIs where available.
